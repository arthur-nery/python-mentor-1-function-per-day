# 🐍 Python Mentor — 1 Function Per Day

**Python Mentor — 1 Function Per Day** is a mentor prompt designed to guide a practical Python learning routine based on the **one-function-per-day method**.

The goal is simple: study Python progressively, one small concept at a time, while turning daily practice into public GitHub documentation, portfolio evidence, and real project-building experience.

---

## 🧠 What Is This Project?

This repository stores a structured prompt used to make an AI act as a technical Python mentor.

The mentor is designed to work as:

- a Python teacher;
- a senior technical mentor;
- a code reviewer;
- a portfolio mentor;
- an operational risk advisor;
- a practical project guide.

Instead of trying to learn many things at once, the prompt enforces one central rule:

> Study only **one function, method, command, or operation per day**.

This helps reduce learning overload and makes progress easier to document, review, publish, and improve over time.

## 🎯 Project Objective

This project was created to document and version the prompt system behind my Python learning journey.

The main objectives are:

1. Preserve the current Python mentor prompt.

4. Build public GitHub documentation from daily practice.
5. Connect Python fundamentals with real-world projects, automation, APIs, files, data, and AI-related workflows.

This is not a traditional Python course.

It is a prompt-based learning system focused on consistency, simplicity, documentation, and portfolio-building.

---

## ⚙️ How It Works

The prompt turns an AI into a strict daily Python mentor.

Each normal daily lesson follows this structure:

1. Review yesterday's function.
2. Teach only one new function, method, command, or operation.
3. Show a minimal example.
4. Explain where it appears in real projects.
5. Show one common mistake.
6. Give one small exercise.
7. Connect the lesson to GitHub or portfolio usage.
8. Mention tomorrow's function without teaching it yet.

The mentor must not skip ahead.

If the topic is large, the mentor must reduce it to the smallest useful function for the day.

---

## 🧩 Megaproject Cycle

Starting in v1.1, the prompt includes a quinzenal Megaproject cycle:

```text
Days 1-14: one function per day
Day 15: Megaproject
Days 16-29: one function per day
Day 30: Megaproject
```

A Megaproject is not meant to increase complexity.

In this project, “mega” means integrating what has already been learned into something simple, useful, and publishable on GitHub.

During a Megaproject day, the mentor must:

- avoid teaching new functions;
- use only previously learned functions, methods, commands, or operations;
- keep the project simple;
- prioritize a GitHub-ready result with a simple README;
- leave future features as improvements if they require concepts not yet studied.

---

## 📌 Core Rule

The central rule of this project is:

> Teach one and only one function, method, command, or operation per day.

Examples:

- Day 1: `print()`
- Day 2: `input()`
- Day 3: one next small concept
- Day 4: one next small concept

The mentor is not allowed to teach the next function before its day.

## 🚀 How To Use

1. Open the prompt file.
2. Copy the full prompt.
3. Paste it into an AI assistant or ChatGPT Project.
4. Start with the command:

```text
Mentor Python, execute
```

5. Follow the daily lesson or Megaproject flow.
6. Write the exercise or project result.
7. Save the code.
8. Commit the progress to GitHub.
9. Repeat the next day.

## 🧭 Recommended Learning Flow

The first mission of the prompt is to create a practical path toward a first public Python project on GitHub.

Initial project idea:

```text
A simple internal control system to register clients, services, and payments.
```

The first lessons start with:

```text
Day 1: print()
Day 2: input()
```

Each new lesson should build slowly toward useful scripts, small automations, and public portfolio projects.

Every 15 days, the Megaproject cycle helps integrate what was already learned into a simple GitHub-ready result.
